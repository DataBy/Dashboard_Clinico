Sistema de Gestión de Clínica  
con Análisis de Rendimiento 
 
Objetivo. 
 
Una clínica privada en crecimiento necesita modernizar su sistema de gestión, por lo 
cual para diseñar este sistema se debe tomar decisiones justificadas sobre qué 
estructuras de datos y algoritmos usar, y demostrarlo con evidencia de cuáles son 
las mejores decisiones. 
 
Datos del sistema. 
 
El sistema maneja tres entidades principales: 
1. Paciente: Cédula, nombre, edad, fecha de registro, nivel de prioridad (1 -
crítico, 2-urgente, 3-normal), tipo de sangre, diagnóstico asignado. 
2. Consulta: ID de consulta, cédula de paciente, fecha, médico tratante, 
diagnóstico, gravedad (1-5), costo. 
3. Diagnóstico: Código, nombre, categoría, subcategoría, descripción. 
 
Para los datos, debe generar conjuntos de distintos tamaños programáticamente, 
por ejemplo: 500, 5000, 50000 y 200000 registros de pacientes. Esto es una 
decisión de diseño desde el inicio, no puede hacerse a mano. 
 
Módulos del sistema.  
 
➢ Módulo 1 — Estructura de atención (Cola con prioridad) 
 
Los pacientes llegan a la clínica y se registran. El sistema debe gestionarlos 
respetando su nivel de prioridad. Se debe decidir si usa una cola simple, una cola 
doble o una cola de prioridad, y justificarlo. La interfaz muestra la lista de espera en 
tiempo real y permite registrar la atención de un paciente. 
 
➢ Módulo 2 — Expedientes y búsquedas 
 
El sistema almacena el historial de consultas de cada paciente. Debe permitir buscar 
por cédula exacta, por nombre (búsqueda parcial), por rango de fechas y por nivel 
de gravedad. Aquí se debe implementar tanto búsqueda lineal como búsqueda 
binaria donde aplique, y la interfaz muestra el tiempo que tomó cada búsqueda para 
que se pueda comparar.

➢ Módulo 3 — Árbol de diagnósticos 
 
Los diagnósticos están organizados jerárquicamente: Área médica → Especialidad 
→ Diagnóstico específico. Por ejemplo: Medicina interna → Cardiología → Infarto 
agudo de miocardio. El sistema debe permitir navegar el árbol, buscar un 
diagnóstico por nombre o código, y mostrar todos los diagnósticos de una 
especialidad. Se debe elegir el tipo de árbol y el tipo de recorrido, y lo justifica. 
 
➢ Módulo 4 — Laboratorio de rendimiento 
 
Desde la interfaz se puede configurar y ejecutar experimentos de rendimiento sobre 
los datos reales del sistema. Funciona así: 
• El usuario selecciona qué conjunto de datos ordenar (pacientes por nombre, por 
edad, por fecha de registro, por prioridad; o consultas por fecha, por gravedad, 
por costo). Luego selecciona uno o varios algoritmos para las comparaciones 
(Bubble Sort, Selection Sort, Insertion Sort, Quick sort). Elige el tamaño del 
conjunto, ejecuta la comparación y la interfaz muestra en tiempo real una gráfica 
de barras con los tiempos, una gráfica de curva de crecimiento al variar el 
tamaño y una tabla con comparaciones estilo historial. 
• Adicionalmente, el módulo permite ejecutar una búsqueda lineal vs una 
búsqueda binaria sobre ese conjunto ya ordenado, midiendo ambos tiempos y 
mostrar en la interfaz los resultados que permita evidenciar cuándo vale la pena 
el costo de ordenar primero.
